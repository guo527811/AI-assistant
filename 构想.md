## 我是郭金明，我想打造一个个人生活助理团队，其中有3个agent角色，作为我的AI分身帮我做一些事情

---

### 1. 新闻秘书 🗞️

**职责：**

- 每天早上定时（建议 8:00）自动搜索 AI / 大模型 / Claude 相关新闻
- 筛选 10-15 条高价值新闻，按重要程度排序
- 用中文总结每条新闻的要点，标注「新技能」「新工具」「新政策」「行业动态」等标签
- 生成结构化的 MD 文档
- 在 `E:\AI助手\新闻简报\2026-06-03\` 下保存（日期按当天）
- 通过邮件发送到个人邮箱

**触发方式：** CronCreate 定时任务（每天早上 8:00）

**工作流：**

1. WebSearch 搜索关键词（AI新闻、大模型、Claude、Anthropic、OpenAI 等）
2. WebFetch 抓取 10-15 篇高价值文章的全文
3. 归纳总结，生成结构化简报
4. 在 E 盘创建日期文件夹，写入 MD 文件
5. 调用邮件 MCP 发送到邮箱

**所需 MCP 工具：** WebSearch、WebFetch、文件系统读写、邮件发送（Gmail MCP / SMTP 脚本）

**所需上下文文件夹：**

- `E:\AI助手\新闻简报\` — 按日期存放每日简报
- `E:\AI助手\配置\关键词.txt` — 新闻搜索关键词列表
- `E:\AI助手\配置\邮箱配置.json` — SMTP 配置（邮箱地址、授权码等）

---

### 2. 学习秘书 📚

**职责：**

- 每天搜索 Claude Code 的最新教程、技巧、案例、官方文档更新
- 整理成结构化学习笔记（概念 → 实操 → 进阶）
- 根据求职目标，标注「面试常考」「实战必会」「了解即可」等优先级
- 每周生成一份学习进度报告
- 数据存到 `E:\AI助手\学习资料\`

**触发方式：** CronCreate 定时任务（每天早上 9:00）+ 手动调用

**工作流：**

1. WebSearch 搜索 Claude Code 教程、Claude API、Anthropic SDK、AI Agent 开发等
2. WebFetch 抓取优质教程/博客/官方文档
3. 提炼核心知识点，生成学习卡片（概念 + 代码示例 + 注意事项）
4. 标注难度等级和求职相关度
5. 存入对应日期的 MD 文件
6. 每周日自动汇总本周学习内容，生成周报

**所需 MCP 工具：** WebSearch、WebFetch、文件系统读写

**所需上下文文件夹：**

- `E:\AI助手\学习资料\每日学习\` — 按日期存放每日学习卡片
- `E:\AI助手\学习资料\学习路线.md` — 总体学习规划（由学习秘书生成和维护）
- `E:\AI助手\学习资料\学习进度.md` — 每周更新进度
- `E:\AI助手\学习资料\资源库.md` — 收藏的好文章/教程/视频链接
- `E:\AI助手\配置\学习偏好.md` — 你的当前水平、学习风格、每天可用时间

---

### 3. 找工作秘书 💼

**职责：**

- 定期扫描招聘市场（Boss直聘、拉勾、LinkedIn、猎聘等）上 "Claude Code" "AI Agent" "大模型应用开发" 相关岗位
- 对比你的简历和岗位 JD，找出技能差距
- 给出简历优化建议 + 项目补充建议
- 将技能差距反馈给学习秘书，动态调整学习计划
- 数据存到 `E:\AI助手\求职\`

**触发方式：** 手动调用或 CronCreate 定时（每周一、周四）

**工作流：**

1. WebSearch 搜索各招聘平台的 Claude Code / AI Agent 相关岗位
2. WebFetch 抓取岗位 JD 详情
3. 解析 JD 中的技能要求、经验要求、薪资范围
4. 读取你的简历，逐条对比技能差距
5. 输出：岗位汇总表 + 技能差距分析 + 简历修改建议 + 学习计划调整建议
6. 如果检测到急缺技能，自动通知学习秘书优先安排

**所需 MCP 工具：** WebSearch、WebFetch、文件系统读写

**所需上下文文件夹：**

- `E:\AI助手\求职\我的简历.md` — 你的最新简历（需提前准备）
- `E:\AI助手\求职\目标岗位.md` — 目标城市、薪资范围、岗位类型、公司规模等偏好
- `E:\AI助手\求职\岗位追踪\` — 按日期存放岗位汇总
- `E:\AI助手\求职\技能差距分析.md` — 持续更新的差距分析
- `E:\AI助手\求职\简历优化建议.md` — 针对性的简历修改建议

## 所需 MCP 工具汇总


| MCP 工具                   | 用途             | 使用者            |
| ------------------------ | -------------- | -------------- |
| **WebSearch**            | 搜索新闻、教程、岗位信息   | 三个秘书           |

## 实施优先级建议

1. **第一优先 — 新闻秘书**：最独立、最可控，能快速跑通「搜索→总结→写文件→发邮件」全流程
2. **第二优先 — 学习秘书**：依赖新闻秘书跑通的经验，复用搜索+总结能力
3. **第三优先 — 找工作秘书**：需要先准备好简历，且依赖学习秘书的技能体系
