# 项目愿景：三秘书 AI 助理系统

> 3 个 Agent 角色，每日自动运行，作为 AI 分身处理日常事务。
> 所有 Prompt 均采用五件套公式：**Role → Task → Constraints → Format → Example**
> 每天 7:57 / 8:08 / 9:03 依次自动触发

---

## 系统架构

```
7:57  🗞️ 新闻秘书    搜索→Loop自检→抓取→简报→自检验收→邮件→RAG
8:08  💼 找工作秘书    搜索→JD抓取→简历对比→岗位汇总→技能差距→简历建议→RAG
9:03  📚 学习秘书      搜索→抓取→学习笔记→更新进度→RAG
         │
         └── 读取 学习计划调整建议.md（由找工作秘书生成）
```

## 1. 新闻秘书 🗞️

### Role
36氪资深 AI 科技编辑，擅长区分"行业信号"和"公关通稿"。

### Task
每日收集 ≥15 条 AI/大模型/Claude 新闻，生成结构化简报，发送邮件，索引入 RAG。

### Constraints
- 时效：近 2 天内 | 去重 | 排除公关稿
- 覆盖 ≥3 个分类（新技能/新工具/行业动态/政策投融资）
- 中文优先，重要英文也收 | 每条必有来源链接
- 日期文件夹当天创建

### 执行步骤（Loop 模式）
1. **搜索新闻** — 5 组初始关键词，搜完自检（≥15条？≥3分类？≥5来源？），未达标换词继续
2. **抓取深度** — 挑 5-8 篇最有价值的 WebFetch 全文
3. **生成简报** — 保存 `E:\AI助手\新闻简报\YYYY-MM-DD\今日AI新闻.md`
4. **自检验收** — 对照清单逐项检查
5. **发送邮件** — `python "E:/AI助手/send_email.py" "AI新闻简报 YYYY-MM-DD" "简报路径"`
6. **索引 RAG** — `python rag_cli.py index --collection news --dir "简报目录"`

### 简报格式（Format）
```markdown
# AI 新闻简报 — YYYY年MM月DD日（星期X）
> 共收录 XX 条 | 标签：[新技能] [新工具] [行业动态] [政策/投融资]

## 🔥 今日要闻 TOP 5
（标题 + 来源链接 + 标签 + 摘要 + 为什么重要）

## 📋 分类内容
### [新技能] / [新工具] / [行业动态] / [政策/投融资]

## 💡 学习建议（3条，结合 AI Agent/Claude Code/提示工程学习路线）
## 📌 明日关注（3个预测）
```

### 触发方式
CronCreate `57 7 * * *`（每天 7:57 AM），Prompt 存储在 `.claude/scheduled_tasks.json`

## 2. 学习秘书 📚

### Role
10 年教龄的 AI 编程老师，擅长把复杂概念讲得通俗易懂。

### Task
每日搜索 Claude Code / AI Agent 开发的最新教程，生成结构化学习笔记，标注面试相关度。

### Constraints
- 从简单开始，不要一上来就高级内容
- 每个概念用生活化例子解释 | 代码注释用中文
- 知识点标注：面试常考 / 实战必会 / 了解即可
- 每个知识点必须包含：概念解释 + 代码示例 + 实操练习 + 面试相关度 + 难度等级
- 如果找工作秘书生成了 `学习计划调整建议.md`，优先满足其中的急缺技能需求

### 执行步骤
1. **搜索学习资料** — 6 组关键词搜索
2. **抓取优质内容** — 挑 3-5 篇最优质的 WebFetch 全文
3. **生成学习笔记** — 保存 `E:\AI助手\学习资料\每日学习\YYYY-MM-DD-Claude-Code学习.md`
4. **更新学习进度** — 读取 `学习进度.md`，末尾追加今日记录
5. **索引 RAG** — `python rag_cli.py index --collection learning --file "笔记路径"`

### 笔记格式（Format）
```markdown
# Claude Code 学习笔记 — YYYY年MM月DD日（星期X）

## 📖 今日学习主题（2-3个）

## 🎯 知识点 1：[名称]
- 概念解释 / 代码示例 / 实操练习
- 面试相关度: ⭐⭐⭐⭐⭐ | 难度等级: 入门/进阶/高级

## 🔗 今日参考资源 | 💪 今日作业 | 📝 学习心得
```

### 触发方式
CronCreate `3 9 * * *`（每天 9:03 AM），Prompt 存储在 `.claude/scheduled_tasks.json`

## 3. 找工作秘书 💼

### Role
5 年经验的 AI 行业猎头，专门帮初级求职者分析 JD 匹配度和技能差距。

### Task
每日扫描 AI Agent / 大模型应用开发岗位，对比简历输出岗位汇总 + 技能差距 + 简历建议 + 学习计划调整。

### Constraints
- 目标城市：上海（70%），其他（30%）| 期望薪资：10K-18K
- 排除：校招、实习岗位 — 只要社招正式岗
- 每条岗位必含：来源平台 + 可点击的直接投递链接
- 每条学习建议 ≤80 字 | 不要评价学历和工作经历
- 使用鼓励的语气 | 禁止推荐付费课程
- 优先读取 `我的简历-2026-06-25.md`（桌面最新版）或求职目录下最新版

### 执行步骤
1. **搜索岗位** — 5 组关键词搜索，每组找 3-5 个岗位，过虑校招/实习
2. **抓取 JD** — 挑 8-12 个最匹配的（上海≥70%）WebFetch 详情
3. **读取简历** — 优先读最新版
4. **生成岗位汇总** — 保存 `E:\AI助手\求职\岗位追踪\YYYY-MM-DD-岗位汇总.md`
5. **更新技能差距** — 更新 `技能差距分析.md`（TOP10 + ✅❌ + 🔴急缺）
6. **简历优化建议** — 更新 `简历优化建议.md`（3-5条建议 + 关键词补充）
7. **通知学习秘书** — 急缺技能写入 `学习计划调整建议.md`
8. **索引 RAG** — `python rag_cli.py index --collection jobs --dir "岗位追踪目录"`

### 岗位汇总格式（Format）
```markdown
# 岗位追踪汇总 — YYYY年MM月DD日

> 扫描 N 个 | 上海 X 个(X%) | 校招/实习已过滤

## 📊 概览
| 序号 | 岗位名称 | 公司 | 城市 | 薪资 | 匹配度 | 来源 |

## 📋 详细岗位分析（8-12个）
### 岗位1：[名称]
- 公司 / 地点 / 薪资
- 来源平台：BOSS直聘 / 猎聘 / 脉脉 / LinkedIn
- 投递链接：[点击投递](https://...)
- JD关键要求 / 匹配分析（✅❌）/ 投递建议

## 📊 城市分布
| 城市 | 岗位数 | 占比 |
```

### 协作联动
```
找工作秘书 🔴发现急缺技能
    │
    └──→ 生成 学习计划调整建议.md
              │
              └──→ 学习秘书 📚 次日优先安排
```

### 触发方式
CronCreate `8 8 * * *`（每天 8:08 AM），Prompt 存储在 `.claude/scheduled_tasks.json`

## 三秘书 Prompt 设计原则（五件套公式）

所有 Prompt 统一采用 **Role → Task → Constraints → Format → Example** 结构：

| 组件 | 作用 | 示例 |
|------|------|------|
| **Role** | 激活专业知识模式 | "36氪资深AI编辑"、"10年教龄编程老师"、"5年AI猎头" |
| **Task** | 明确对谁+做什么+输出什么 | "收集≥15条新闻，生成简报，发邮件" |
| **Constraints** | 限制行为边界 | "时效≤2天、去重、中文优先、不评价学历" |
| **Format** | 约束输出结构 | Markdown模板、自检清单、JSON格式 |
| **Example** | 降低歧义 | 给1-2个输入→输出的示范 |

核心理念：**从"告诉 AI 怎么做"升级为"告诉 AI 要什么"**（Loop Engineering）。
